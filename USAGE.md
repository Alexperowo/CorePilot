# Как пользоваться CorePilot

Каждый режим — на своей вкладке в окне приложения (интерфейс DearPyGui, без браузера). Перед началом убедитесь, что во вкладке **⚙ Настройки** выбраны рабочие модели для ролей (см. [CONFIGURATION.md](CONFIGURATION.md)). Общее состояние фабрики видно на вкладке **📊 Обзор**.

---

## 🛠 Режим «Конвейер» — работа с файлами (код, документы, шаблоны)

Интерактивный режим: вы пишете текстовый запрос, и команда специализированных агентов выполняет его за 4 шага. Подходит как для программирования, так и для рутинной работы с документами, текстами, таблицами и шаблонами.

**Как работает:**
1. **Сборщик** — изучает файлы в папке проекта, собирает контекст.
2. **Архитектор** — строит пошаговый план изменений.
3. **Кодер (Исполнитель)** — вносит изменения в файлы (пишет код, правит тексты, заполняет шаблоны).
4. **Аудитор** — проверяет результат работы на соответствие исходному запросу.

После этого вы увидите **diff** (разницу до/после) и кнопки **Применить / Отклонить**. Ничего в файлах проекта не изменится без вашего явного согласия (если не включен тумблер авто-применения).

**Примеры запросов:**
* *Для кода:* «Добавь обработку ошибок в функцию загрузки файла в utils.py»
* *Для документов и рутины:* «Заполни пустой шаблон отчёта report.md данными из логов», «Распарси csv-файл с контактами и приведи телефонные номера к формату +7...», «Напиши ТЗ по ГОСТу на основе плана в roadmap.txt».

**Совет:** чем конкретнее запрос и чем точнее указаны файлы, тем точнее и быстрее ИИ-агенты справятся с задачей.

## 📋 Режим «Менеджер» — постановка длительных задач

Превращает крупную составную цель в список конкретных задач (DAG-граф) и складывает их в очередь для автономного фонового Демона. Идеально для длительных процессов автоматизации.

**Как работает:**
1. Вы вводите общую цель (например, провести аудит всей папки с документами или покрыть проект тестами).
2. **Product Owner** (агент) формирует общую концепцию решения.
3. **Scrum Master** (агент) разбивает концепцию на отдельные связанные подзадачи (JSON-бэклог).
4. Задачи сохраняются в папку `auto_tasks\` и отображаются в статистике очереди.

**Запуск Демона.** В этом режиме есть кнопки **🚀 Запустить Демона** / **🛑 Остановить**. Демон работает в фоне, по одной берёт задачи из очереди и прогоняет их через конвейер (Сборщик → Архитектор → Кодер → Аудитор) полностью автономно.

- Успешно выполненные задачи уходят в `auto_tasks\done\`.
- Задачи со сбоями — в `auto_tasks\failed\` (с детальным описанием ошибки; при повторном сбое на помощь приходит **Оракул** с интеллектуальной подсказкой).

**Примеры целей:**
* *Для кода:* «Покрой модуль обработки платежей юнит-тестами»
* *Для документов и рутины:* «Проверь все файлы отчётов в папке docs, найди орфографические ошибки, исправь форматирование таблиц в markdown и заполни пустые ячейки исполнителей»

## 🧹 Режим «AI Cleaner» — очистка диска

Безопасный анализ и очистка. **Ничего не удаляется навсегда сразу** — всё проходит через карантин.

**Кнопки сканирования:**
- **🧹 Глубокий кэш** — кэши приложений в LOCALAPPDATA.
- **👻 Остатки программ** — папки на диске C.
- **📥 Анализ загрузок** — крупные/старые файлы в «Загрузках».
- **👯 Найти дубликаты** — одинаковые файлы (подтверждаются полным хэшем).

**Как работает:**
1. Сканеры (быстрый детерминированный код, без ИИ) собирают объекты.
2. Агент-аналитик классифицирует риск каждого: ✅ безопасно / ⚠️ внимание / 🚨 не трогать.
3. Вы видите отчёт с размерами и кнопки удаления по уровням риска.

**Карантин:**
- Удаление = перемещение в карантин (на тот же диск).
- Кнопка **↩️ Отменить** возвращает файлы.
- Кнопка **💀 Удалить навсегда** очищает карантин окончательно.
- **📦 Статус карантина** — управление всеми сессиями.

> Системные папки (Windows, Program Files, system32) и элементы автозапуска **никогда** не попадают под автоудаление — только в категорию «не трогать».

## Запуск локальных моделей (менеджер моделей)

В портативной версии за запуск моделей отвечает интерактивный менеджер (`start_llama.bat` → `llama_manager.py`). При запуске он по шагам спросит:

1. **Модель** — выбор из `.gguf` файлов в папке `models\`.
2. **Размер контекста** (`-c`) — сколько текста модель «держит в голове». По умолчанию 4096 (безопасно для 8 ГБ VRAM; для систем с запасом VRAM можно увеличить).
3. **Сжатие KV-кэша** — экономит видеопамять. `q8_0` — безопасный выбор (почти без потерь). Ниже `q5_0` логика модели начинает страдать — берите только при острой нехватке VRAM. При сжатии автоматически включается Flash Attention.
4. **Слои на GPU** (`-ngl`) — менеджер оценит вашу видеопамять и предложит оптимальное число. Согласитесь или введите своё (99 = вся модель на GPU, меньше = часть на CPU при нехватке VRAM).

После подтверждения сервер запустится на порту 8080. В настройках CorePilot выберите локальный бэкенд `llamacpp`.

## Общие советы

- **Одна задача за раз.** Пока система занята, новые сообщения отклоняются — дождитесь завершения.
- **Локальные модели и память.** При 8 ГБ VRAM и разных моделях на роли включите выгрузку VRAM между агентами (см. конфигурацию).
- **Генерация изображений.** В режиме «Менеджер» ключевые слова вроде «нарисуй» / «сгенерируй» запускают генерацию. Источник выбирается в настройках (группа «Генерация изображений»):
  - **SD Forge** (локальный, порт 7860) — нужен запущенный Forge с флагом `--api`
  - **ComfyUI** (локальный, порт 8188) — нужен запущенный ComfyUI
  - **Облако** — поддерживаются: `huggingface` (бесплатно, `HF_TOKEN`), `openai` (DALL-E 3, `OPENAI_API_KEY`), `together` (FLUX, `TOGETHER_API_KEY`), `stability` (SD3, `STABILITY_API_KEY`), `fal` (`FAL_API_KEY`), `replicate` (`REPLICATE_API_KEY`)

  После генерации автоматически применяется **нейросетевой апскейл 4×** через Forge R-ESRGAN (512px → 2048px). Масштаб и включение настраиваются. Если Forge недоступен — апскейл через Replicate. Если выбран локальный источник, но он недоступен, а облачный ключ есть — автоматический фолбэк в облако. Готовые изображения сохраняются в папку `generated_images/`.
