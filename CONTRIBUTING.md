# Как помочь проекту CorePilot

Спасибо за интерес к проекту! CorePilot — русскоязычный Windows-проект на Python 3.12.

## Быстрый старт

```bat
git clone https://github.com/Alexperowo/CorePilot.git
cd CorePilot
setup.bat          REM создаёт venv и ставит зависимости
python tests/run_all.py  REM убедитесь, что все тесты зелёные
```

## Требования

- **Python 3.12** (строго — из-за `tomllib`, синтаксиса `match/case`, `type X = ...`)
- **Windows 10/11** — основная платформа
- **GPU с Vulkan или CUDA** — для тестирования локального инференса (опционально)

## Архитектура проекта

Описана в `CLAUDE.md`. Ключевое правило — **изоляция слоёв**:
UI (`ui_dpg.py`, `ui_tabs.py`) общается с логикой **только через `service_layer.py`**.
Всё платформенно-зависимое для Android — только в `mobile/android_bridge.py`.

Перед правкой: прочтите `CLAUDE.md` и `NOTES_LEARNINGS.md` (история решений и выстраданных багов).

## Рабочий процесс

1. Форкните репозиторий, создайте ветку с понятным именем (`fix/json-parser`, `feat/vram-presets`).
2. Правьте код. Тесты: `python tests/run_all.py` — должно быть **37/37+** зелёных.
3. Запустите `diagnose.py` — убедитесь, что окружение в порядке.
4. Откройте Pull Request с описанием: что изменено и почему.

## Правила

- Никакого хардкода путей, IP, ключей, параметров железа — всё настраивается.
- Переводы строк: `.py` → LF, `.bat`/`.toml`/`.md` → CRLF (см. `.gitattributes`).
- **Никогда не коммитьте** `secrets.toml`, `user_data/`, `*.gguf`, `venv/` — они в `.gitignore`.
- Новые фичи — под тот же принцип «работает на любом железе»: без предположений об объёме VRAM.

## Вопросы и баги

Используйте GitHub Issues — шаблоны подскажут нужные данные (ОС, видеокарта, VRAM, логи).
