## ТРИ ГЛАВНЫХ ПРИНЦИПА ПРОЕКТА (важнее всего остального)

1. **Минимальное железо — это НИЖНЯЯ планка, а не требование.** Машина Александра
   (16 ГБ RAM, RX 6600 8 ГБ VRAM, Windows 10) — это МИНИМУМ, на котором всё обязано
   работать. Проект должен так же работать и на ДРУГОМ железе: больше или меньше VRAM,
   другие видеокарты, слабее/мощнее. Никаких допущений «у пользователя ровно столько-то».
   Всё, что зависит от железа (VRAM, контекст, число слоёв на GPU) — определяется
   автоматически или настраивается, с разумными значениями по умолчанию.
2. **Понятность для неопытного пользователя.** Целевой пользователь — НЕ программист.
   Любая функция должна быть понятна без чтения кода: ясные подписи, понятные сообщения
   об ошибках («нет доступа к папке моделей — выдайте разрешение», а не пустой экран),
   разумные значения по умолчанию, чтобы заработало «из коробки».
3. **Универсальность.** Никакого хардкода — ни путей, ни IP, ни ключей, ни параметров
   железа. Всё настраивается. Новые фичи делаются гибкими, не под один конкретный случай.

Эти три принципа — критерий любого решения. Если правка нарушает хоть один — переделай.

## ЧТО ТАКОЕ COREPILOT

Локальная мультиагентная «фабрика ИИ» для Windows: команда ИИ-агентов на железе
пользователя — бесплатно (локальные модели) или через бесплатные облачные API.

**Стек:** Python 3.12, без WSL2/Docker. Десктоп UI — DearPyGui. Мобильное — Flet (Android).
Агенты — CrewAI. Ключевые версии пинованы в `requirements.txt` / `mobile/pyproject.toml`
(особо: `flet==0.80.5` — символ `<` в версии ломает сборку APK; не трогай без причины).

**Три режима:**
1. **Конвейер** — правка кода: сбор контекста → план → правка → аудит.
2. **Менеджер + Демон** — DAG-бэклог задач, фоновый Демон выполняет по зависимостям
   (сценарий «оставил на ночь — утром готово»).
3. **AI Cleaner** — очистка системы (Windows).

**5 ролей агентов** (`gatherer/architect/coder/auditor/oracle` — Сборщик/Архитектор/Кодер/
Аудитор/Оракул): у каждой свой бэкенд и модель (разные модели снижают галлюцинации).

**Бэкенды моделей:** локальные (LM Studio, Ollama, llama.cpp, Lemonade) и облако.
**Облачные провайдеры:** бесплатные (Groq, OpenRouter, Cerebras, SambaNova, HuggingFace,
Cohere) — приоритет для простого пользователя; плюс платные (OpenAI, Gemini, Anthropic,
Mistral, DeepSeek). Ключи — в `secrets.toml`, можно несколько на провайдера (ротация).

## АРХИТЕКТУРА (не нарушать)

- **Изоляция слоёв:** UI (`ui_*.py`, `mobile/main.py`) общается ТОЛЬКО через `service_layer.py`.
  UI не лезет в `agents`/`daemon`/логику напрямую.
- **Zero Trust к выводу LLM:** любой ответ модели валидируй, не верь формату (особенно
  локальные модели — см. баг с JSON ниже).
- **Без хардкода путей:** пути от `__file__`, не от текущей папки.
- **Доступность:** Александр слабовидящий — в UI крупный шрифт, высокий контраст.
- **Платформозависимое — изолировано:** в мобильном всё про Android (права, пути, TTS) — в
  `android_bridge.py`; остальной код кроссплатформенный. В десктопе ASCII-маркеры в файлах/
  консоли вместо эмодзи (кодировки Windows).

**Сердце:** `service_layer.py` (UI↔логика: профили, настройки, очередь, модели),
`agents.py` (LLM, ротация ключей, отказоустойчивый `safe_kickoff` с fallback),
`daemon.py` (фоновый исполнитель), `llama_manager.py` (встроенный llama.cpp-сервер),
`tools.py` (инструменты + генерация картинок forge/comfy/cloud). Мобильное:
`mobile/main.py` + `mobile/cloud_api.py` (ротация ключей, urllib — не requests).

**Хранение данных:** профили → `user_data/config_profiles.json` (атомарная запись, НЕ в
`auto_tasks`); очередь → `auto_tasks/`; БД → `factory_data.db`; настройки сессии — десктоп
`.ai_session.json`, мобильное `client_storage` (не публичная папка!); секреты → `secrets.toml`
(НИКОГДА не выводить ключи в чат/лог). `user_data/`, `secrets.toml`, `*.processing` — в gitignore.

## ЖУРНАЛ КЛЮЧЕВЫХ БАГОВ (НЕ ревизовать — выстрадано)

Подробности — в `NOTES_LEARNINGS.md`. Кратко, чтобы не наступить повторно:

- **Профили DPG:** combo с фиксированными строковыми тегами при пересоздании не оживают
  (зомби-алиасы). Решение: `dpg.generate_uuid()` + пересоздание контейнера через
  `delete_item(children_only=True)`.
- **Профили «исчезали»:** демон сканировал все `*.json` в `auto_tasks` и принимал файл
  профилей за задачу. Решение: профили в `user_data/`, демон фильтрует системные файлы,
  атомарная запись + восстановление застрявших `.processing`.
- **Отказоустойчивость провайдеров:** RLock от дедлока ключей; межпровайдерный fallback в
  `safe_kickoff` (упал провайдер → резервный → локальная модель); `peek_api_key` не двигает
  ротацию; обход бага CrewAI #5886 (`cache_breakpoint` чистится для всех кроме anthropic);
  `response_format=json` выключен для локальных (LM Studio его отвергает).
- **Память/llama.cpp:** наш сервер с `--no-mmap` (иначе копия модели в RAM); контекст по
  умолчанию 4096 для 8 ГБ VRAM. Определение VRAM на Windows занижено (WMI) — есть ручной
  ввод. Ollama порт 11434.
- **Мобильное:** настройки в `client_storage` (публичная папка → PermissionError на старте);
  crash logger в `main()`; сборка APK чинена (лицензии SDK, PATH, точная версия flet).

**Открытый баг (не решён):** таймаут этапа Демона через ThreadPoolExecutor не убивает
зависший поток (Python не умеет убивать треды) — модель висит в VRAM. Полное решение —
переписать воркеры на `multiprocessing` с `terminate()`. Крупный рефакторинг, GitHub Issue #6.

## АКТИВНЫЕ ЗАДАЧИ (текущий фокус)

1. **Мобильное не видит модели** — нет запроса прав на хранилище (модели в `Download/CorePilot/
   models`, на Android 11+ нужно разрешение; сейчас запрашивается только микрофон). Чинить:
   запрос `MANAGE_EXTERNAL_STORAGE` после старта UI, либо вариант с внутренней папкой
   приложения (без прав). При ошибке доступа — понятное сообщение, не пустой список.
2. **Невалидный JSON на локальных моделях** — облако ОК, локальные периодически дают
   невалидный JSON (текст вокруг JSON, markdown-обёртка). Усилить парсер (вырезать обёртку,
   брать JSON из текста, ретраи), few-shot в промпты, рассмотреть GBNF-грамматику для нашего
   сервера.
3. **install_apk.bat** перестал работать (путь к APK сменился — Flet кладёт в свою папку).
4. Полировка трёх режимов (Cleaner, Конвейер, Демон), затем доводка по `TESTPLAN.md`.

Отлаживать мобильное — на виртуальном Android (эмулятор + десктоп на одной машине). Детали —
в `ROADMAP_DEV.md` и `MOBILE_DESKTOP_TEST.md`.
