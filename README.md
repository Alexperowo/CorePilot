# CorePilot (AI Factory) 🏭

**CorePilot** — это локальная фабрика ИИ-агентов для работы с кодом, документами и операционной системой. Несколько специализированных ИИ-агентов взаимодействуют как автономная команда для решения ваших задач непосредственно на вашем компьютере с использованием локальных или облачных моделей.

> Работает на обычном железе. Проверено на Ryzen 5 3600 / 16 ГБ RAM / Radeon RX 6600 8 ГБ — без дискретной видеокарты тоже запустится (медленнее).

---

## 🌟 Философия проекта

**CorePilot** спроектирован как **автономный ИИ-оркестратор длительных задач** с интеллектуальным управлением контекстом.

* 💻 **Локальная эффективность и гибридность**: Оптимизирован для эффективной работы локальных моделей на нетребовательном оборудовании и гибкой интеграции в гибридные ИИ-инфраструктуры (совместное использование локального инференса и облачных API).
* 👶 **Доступность и простота**: Понятен для неопытных и простых пользователей. Стабилен, автономен и не требует сложной ручной настройки.
* 📋 **Решение рутинных задач (не только кодинг)**: 
  * Помощь в работе с простыми документами, отчётами и текстами.
  * Умный парсинг данных, заполнение строк, шаблонов и форм.
  * Автоматизация повседневной рутинной работы за ПК.
  * **Умная очистка и оптимизация**: встроенный модуль *AI Cleaner* проводит анализ и оптимизацию ПК умнее и безопаснее классических шаблонных утилит за счет работы ИИ-агентов и механизма временного карантина с возможностью отмены.

## Что умеет CorePilot

Графический интерфейс (DearPyGui, без браузера) состоит из вкладок. Основные режимы работы:

| Режим | Иконка | Что делает |
|---|---|---|
| **Конвейер** | 🛠 | Интерактивно правит код по вашему запросу: собирает контекст → планирует → пишет → проверяет. Показывает diff перед применением. |
| **Менеджер** | 📋 | Разбивает большую цель на DAG-граф задач (с зависимостями) и складывает их в очередь для фонового Демона. |
| **AI Cleaner** | 🧹 | Безопасно анализирует и чистит диск: кэши, дубликаты, старые загрузки. Всё удаляемое сначала уходит в **карантин** (можно вернуть). |

Плюс вкладки центра управления: **📊 Обзор** (дашборд здоровья фабрики), **Граф DAG** (визуализация зависимостей задач с авто-раскладкой), **Канбан**, **🦙 Llama-сервер** (запуск локальных моделей), **🧪 Auto QA** (стресс-тесты ядра), **📜 Лог демона**, **⚙ Настройки** (каскад 5 ролей + профили конфигурации).

Интерфейс рассчитан и на пользователей со слабым зрением: крупный шрифт (регулируется), высокий контраст, статусы дублируются текстом и символами, опциональные звуковые сигналы.

**Мобильное приложение-компаньон (Android).** В папке `mobile/` — компаньон на Flet. Три способа работы: лёгкий Edge-AI чат с локальной моделью прямо на телефоне (движок запускается из `nativeLibraryDir` — единственный исполняемый путь на нерутованном Android), подключение к десктопной фабрике по IP, и автономный чат-бот через бесплатные облачные API (HuggingFace, OpenRouter, Cerebras, Groq, OpenAI) с ротацией ключей и генерацией изображений. Системный промпт, температура и лимит токенов настраиваются. Это спутник десктопной фабрики для задач на ходу.

## Быстрый старт (портативная версия) — рекомендуется

Это самый простой путь: ничего не нужно устанавливать в систему.

1. **Соберите портативную версию.** Дважды кликните `Portable.bat`. Скрипт сам скачает Python, зависимости и сервер моделей `llama.cpp`.
   - Когда спросит про бэкенд — для Radeon RX 6600 (и любых AMD/Intel) выбирайте **[1] Vulkan**.
   - Для NVIDIA — **[2] CUDA 12** или **[3] CUDA 11** (по версии драйверов). Без видеокарты — **[4] CPU**.
2. **Положите модель.** Скачайте любую GGUF-модель (например, с Hugging Face) и положите `.gguf`-файл в папку `CorePilot_Portable\models\`.
3. **Запустите.** Дважды кликните `CorePilot_Portable\START_HERE.bat`. Откроется графический интерфейс (отдельное окно, без браузера).

Подробности — в [INSTALL.md](INSTALL.md).

## Обычная установка (стационарная)

Если Python 3.12 уже установлен — два двойных клика:

1. `setup.bat` — создаёт окружение и ставит зависимости (один раз).
2. `start.bat` — запускает графический интерфейс.

Либо вручную:

```bat
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python ui_dpg.py
```

Откроется нативное окно интерфейса (DearPyGui, рендер через DirectX, без браузера).

> Запасной веб-интерфейс (Chainlit) остаётся доступен: раскомментируйте `chainlit`
> в `requirements.txt`, установите его и запустите `chainlit run app.py -w`
> (откроется `http://localhost:8000`).

## Настройка моделей (главное)

Откройте вкладку **⚙ Настройки** в интерфейсе. Вверху — каскад из 5 ролей: **Сборщик, Архитектор, Кодер, Аудитор, Оракул**. Для каждой роли по очереди выбираете:

1. **Источник** — `local` (модель на вашем ПК) или `cloud` (через API).
2. Если `local` → **бэкенд** (LM Studio, Ollama, LlamaCPP, Lemonade) → **модель**.
3. Если `cloud` → **провайдер** (бесплатные: Groq, OpenRouter, Cerebras, SambaNova, HuggingFace, Cohere; платные: Anthropic, OpenAI, Gemini, Mistral, DeepSeek) → **модель**.

> **Совет для 8 ГБ VRAM.** Можно назначить разным ролям разные локальные модели (например, специализированную модель-кодер для Кодера) — это снижает галлюцинации. Чтобы две модели не занимали память одновременно, включите тумблер **«Выгружать модель из VRAM между агентами»**.

Локальный сервер `llama.cpp` запускается прямо из интерфейса на вкладке **🦙 Llama-сервер** (выбор модели, контекст, KV-кэш, автоподбор `-ngl` по VRAM). Облачные ключи кладутся в `secrets.toml` — см. [CONFIGURATION.md](CONFIGURATION.md).

## Структура проекта

```
CorePilot/
├─ ui_dpg.py           # ОСНОВНОЙ интерфейс (DearPyGui): точка входа, окно, цикл
├─ ui_tabs.py          #   — классы всех 10 вкладок
├─ ui_common.py        #   — общая основа: палитра, тема, шрифты, фоновые потоки
├─ service_layer.py    # Тонкий фасад UI↔ядро (UI знает только его)
├─ app.py              # Запасной веб-интерфейс (Chainlit)
├─ daemon.py           # Фоновый Демон: берёт задачи из очереди (DAG) и выполняет
├─ agents.py           # Фабрика LLM по ролям, устойчивый запуск, квоты, выгрузка VRAM
├─ router.py           # Авто-выбор профиля под задачу
├─ pipeline_*.py       # Режим «Конвейер» (агенты + поток + парсер)
├─ manager_agents.py   # Режим «Менеджер» (Product Owner + Scrum Master, DAG-бэклог)
├─ cleaner_*.py        # Режим «AI Cleaner» (сканеры + карантин)
├─ llama_manager.py    # Запуск локального сервера llama.cpp (+ CLI-режим)
├─ tools.py            # Инструменты агентов (файлы, поиск, терминал, генерация картинок: Forge/ComfyUI/облако)
├─ utils.py            # SessionState, песочница, чекпойнты, утилиты
├─ context_manager.py  # SQLite: логи, задачи, история очистки
├─ auto_qa.py          # Автономный стресс-тестировщик ядра (5 режимов)
├─ field_test.py       # Сквозной полевой тест (10 фаз, headless)
├─ tests/              # Юнит-тесты ядра (pytest либо python tests/run_all.py)
├─ user_data/          # Профили и пользовательские данные (вне репозитория)
├─ setup.bat           # Стационарная установка: venv + зависимости + llama.cpp (один раз)
├─ start.bat           # Запуск интерфейса (стационарная версия)
├─ update_llama.bat    # Обновление движка llama.cpp до последней версии
├─ Portable.bat        # Сборщик портативной версии (Python + llama.cpp в комплекте)
├─ CLAUDE.md           # Технический профиль проекта (для разработки в Claude Code)
├─ mobile/             # Мобильное приложение-компаньон (Flet, Android)
│   ├─ main.py         #   — интерфейс компаньона (Edge AI, чат, логи, настройки)
│   ├─ cloud_api.py    #   — облачные провайдеры + ротация ключей (текст/картинки)
│   ├─ android_bridge.py  # — мост к движку на Android (nativeLibraryDir)
│   ├─ llama_server.py #   — локальный llama-server на телефоне
│   └─ build_apk.bat   #   — сборка APK (лицензии, PATH, Flutter автоматически)
└─ requirements.txt
```

## Безопасность

- **Песочница (по умолчанию вкл.).** Конвейер пишет изменения во временную копию; в реальный проект они попадают только после вашего подтверждения diff.
- **Карантин очистки.** AI Cleaner ничего не удаляет сразу — файлы перемещаются в карантин на том же диске, откуда их можно восстановить одной кнопкой.
- **Защита системных путей.** Windows, system32, Program Files и автозапуск никогда не трогаются автоматически.
- **Подтверждение команд.** Выполнение терминальных команд в Конвейере требует вашего согласия; в автономном Демоне терминал заблокирован. Команды выполняются через системную оболочку — не запускайте инструмент на недоверенных задачах без присмотра (см. «Известные ограничения» в TROUBLESHOOTING.md).
- **Маскирование секретов.** API-ключи автоматически редактируются в логах и трейсах ошибок.
