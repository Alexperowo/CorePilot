# Установка CorePilot

Два способа: **портативный** (проще, ничего не ставится в систему) и **обычный** (для тех, у кого уже есть Python).

---

## Способ 1. Портативная версия (рекомендуется новичкам)

### Что нужно заранее
- Windows 10 или 11 (64-битная).
- Подключение к интернету (только на время сборки).
- ~3–5 ГБ свободного места (Python + зависимости + сервер моделей; модель отдельно).

### Шаги

**1. Запустите сборщик.**
Дважды кликните файл `Portable.bat`. Откроется чёрное окно.

**2. Выберите бэкенд для моделей.**
Скрипт спросит, какой ускоритель использовать:
- **1 — Vulkan** — подходит для видеокарт AMD Radeon (включая RX 6600), Intel Arc, а также NVIDIA. Если сомневаетесь — выбирайте этот.
- **2 — CUDA 12** — для NVIDIA с новыми драйверами (CUDA 12).
- **3 — CUDA 11** — для NVIDIA со старыми драйверами (CUDA 11).
- **4 — CPU** — без видеокарты. Работает везде, но медленно.

Введите цифру и нажмите Enter.

**3. Дождитесь окончания.**
Скрипт автоматически:
- скачает портативный Python;
- установит все зависимости;
- скачает последнюю версию сервера `llama.cpp` под выбранный бэкенд;
- создаст ярлыки запуска.

Это занимает несколько минут (зависит от интернета).

**4. Добавьте модель.**
Скачайте GGUF-модель (см. раздел «Где взять модель» ниже) и положите `.gguf`-файл в папку:
```
CorePilot_Portable\models\
```

**5. Запустите.**
Дважды кликните:
```
CorePilot_Portable\START_HERE.bat
```
Откроется графический интерфейс CorePilot — отдельное окно, без браузера.

Локальный сервер моделей запускается **внутри интерфейса**, на вкладке **🦙 Llama-сервер**: выберите модель из `models\`, размер контекста, тип сжатия KV-кэша и число слоёв на GPU (`-ngl`). Интерфейс сам оценит видеопамять и предложит значение (или поставьте `-ngl = -1` для автоподбора). Сервер поднимется на порту 8080.

> Дополнительные батники: `start_llama.bat` — интерактивный CLI-менеджер моделей (запасной), `start_chainlit.bat` — запасной веб-интерфейс в браузере (нужен установленный `chainlit`).

## Способ 2. Обычная установка

### Что нужно заранее
- **Python 3.12** ([скачать](https://www.python.org/downloads/)). При установке отметьте галочку **«Add Python to PATH»**.
- Движок локальных моделей **устанавливать заранее не нужно** — `setup.bat` сам поставит `llama.cpp` server (вы выберете Vulkan/CUDA/CPU). Сторонние серверы [LM Studio](https://lmstudio.ai/) или [Ollama](https://ollama.com/) — опционально, для продвинутых (выбираются в настройках приложения). Либо ключи облачных API.

### Шаги

**Вариант с батниками (рекомендуется — без командной строки):**

1. Дважды кликните `setup.bat`. Он: создаст виртуальную среду, установит зависимости и **скачает llama.cpp server** под выбранный бэкенд (Vulkan для AMD/Intel/NVIDIA, CUDA для NVIDIA, или CPU). Запускается один раз.
2. Дважды кликните `start.bat` — откроется графический интерфейс.

> Бинарник сервера кладётся в папку `llama\` рядом с проектом. Если выбрать «Пропустить», локальные модели будут работать только через LM Studio/Ollama.

**Вариант вручную (командная строка):**

```bat
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python ui_dpg.py
```

> При ручной установке llama.cpp server не скачивается — используйте `setup.bat` для автоматической загрузки, либо скачайте `llama-server` вручную с [релизов llama.cpp](https://github.com/ggml-org/llama.cpp/releases/latest) в папку `llama\`.

> **Запасной веб-интерфейс.** Если предпочитаете старый Chainlit: раскомментируйте `chainlit` в `requirements.txt`, повторно запустите `setup.bat`, затем `start_chainlit.bat` (или вручную `chainlit run app.py -w`) — откроется `http://localhost:8000`.

## Где взять модель (GGUF)

GGUF — это формат локальных моделей. Скачать можно с [Hugging Face](https://huggingface.co/models?library=gguf).

Под 8 ГБ видеопамяти хорошо подходят модели **7–8B** в квантовании **Q4_K_M** (занимают ~4.5–5 ГБ). Примеры семейств: Qwen2.5-Coder, Llama 3.1/3.2, DeepSeek-Coder, Mistral.

> Совет: для роли «Кодер» берите модель, заточенную под код (в названии есть «Coder» / «Code»).

---

## Проверка, что всё работает

1. В интерфейсе откройте ⚙️ настройки.
2. Вверху для роли **Кодер** выберите источник `local`, бэкенд (например, `llamacpp` для портативной версии — порт 8080) и вашу модель из списка.
3. Сохраните. Перейдите в режим **🛠 Конвейер** и напишите простой запрос, например: «покажи структуру проекта».

Если агент ответил — установка успешна. Если что-то пошло не так, см. [TROUBLESHOOTING.md](TROUBLESHOOTING.md).

## Обновление движка llama.cpp

llama.cpp обновляется часто. Чтобы обновить движок локальных моделей до последней версии:

1. Дважды кликните `update_llama.bat`.
2. Выберите тот же бэкенд, что при установке (Vulkan / CUDA / CPU).

Скрипт скачает свежую версию, а старую сохранит в `llama\backup_ГГГГММДД` — если новая заработает, папку backup можно удалить. Работает и для стационарной (`llama\`), и для портативной (`CorePilot_Portable\llama\`) установки — нужная папка находится автоматически.

## Мобильное приложение-компаньон (Android)

В папке `mobile/` — компаньон на Flet: локальный Edge-AI чат прямо на телефоне, просмотр логов, настройки. Это спутник десктопной фабрики для лёгких задач на ходу. Сборка APK выполняется через Flet (`flet build apk`); подробности — в комментариях `mobile/main.py` и `mobile/android_bridge.py`. Десктопная версия остаётся основной — мобильная её не заменяет.

## Важно для систем с 16 ГБ RAM (управление памятью)

Если используете **LM Studio** как локальный бэкенд: в настройках загрузки модели
снимите галочку **«Use mmap (Keep model in RAM)»**. Иначе модель занимает память
дважды (в VRAM и в RAM одновременно), и на 16 ГБ система уходит в своп. Подробнее —
в TROUBLESHOOTING.md, раздел про оперативную память.

Встроенный llama.cpp-сервер CorePilot уже настроен правильно (`--no-mmap`, контекст
4096), дополнительных действий не требует.
