# Решение проблем

## Установка и запуск

**`Portable.bat` не скачивает Python / llama.cpp**
Проверьте интернет и антивирус (иногда блокирует PowerShell-загрузки). Можно переопределить версию Python: задайте переменную `PY_VERSION` перед запуском. Если не качается llama.cpp — скачайте архив вручную со [страницы релизов](https://github.com/ggml-org/llama.cpp/releases/latest) (нужен `win-vulkan-x64` или `win-cuda-*`) и распакуйте в `CorePilot_Portable\llama\` так, чтобы там оказался `llama-server.exe`.

**Окно интерфейса не открывается**
Убедитесь, что установлен `dearpygui` (`pip install dearpygui`). На очень старых видеодрайверах DearPyGui может не инициализировать DirectX — обновите драйвер GPU. Текст в окне видно через `python ui_dpg.py` в консоли (ошибки печатаются туда). Если нужен браузерный интерфейс — используйте запасной Chainlit (`start_chainlit.bat`, требует установленного `chainlit`).

**`ModuleNotFoundError` при обычной установке**
Активирована ли виртуальная среда? Выполните `venv\Scripts\activate`, затем повторно `pip install -r requirements.txt`.

## Модели

**В списке моделей пусто / «бэкенд офлайн»**
Сервер моделей не запущен. Для портативной версии запустите `start_llama.bat`. Для LM Studio/Ollama — запустите их и убедитесь, что модель загружена. Проверьте, что URL и порт в настройках совпадают с сервером (см. [CONFIGURATION.md](CONFIGURATION.md)).

**Агент падает с ошибкой «Не выбрана локальная модель»**
В настройках для этой роли источник стоит `local`, но модель не выбрана (сервер был офлайн при открытии настроек). Запустите сервер моделей, переоткройте настройки, выберите модель, сохраните.

**Модель отвечает очень медленно**
Скорее всего, работа идёт на CPU. Проверьте: в окне менеджера моделей (`start_llama.bat`) при выборе `-ngl` менеджер показывает обнаруженную видеопамять; в логе сервера должно быть видно, что определился GPU (Vulkan/CUDA). Если VRAM мало (8 ГБ) и модель крупная — при запросе `-ngl` введите меньшее число слоёв (например, 20 вместо предложенного), либо выберите сжатие KV-кэша q8_0.

**Не удаётся скачать нужный бэкенд / выбрал не ту версию CUDA**
При сборке `Portable.bat` предлагает: Vulkan, CUDA 12, CUDA 11, CPU. Для NVIDIA версия CUDA зависит от драйверов — если CUDA-сборка не стартует, попробуйте другую версию или Vulkan (работает на NVIDIA тоже). Архив можно скачать вручную со [страницы релизов](https://github.com/ggml-org/llama.cpp/releases/latest) и распаковать в `CorePilot_Portable\llama\`.

**Нехватка видеопамяти (out of memory) при разных моделях на роли**
Включите тумблер **«Выгружать модель из VRAM между агентами»** в настройках. Он освобождает память предыдущего агента (на Ollama) перед следующим.

## Облачные провайдеры

**`Cloud LLM init failed` / нет ответа от облака**
Проверьте `secrets.toml`: правильное имя ключа (`GROQ_API_KEY` и т.п.), ключ действителен, есть квота. Файл должен лежать рядом с кодом проекта (или в `.chainlit/secrets.toml`). Остаток квот можно посмотреть на вкладке **📊 Обзор** → «Обновить квоты».

**Постоянные ошибки лимита (429)**
Добавьте несколько ключей через запятую (`GROQ_KEYS = "k1,k2"`) — система будет чередовать их. Либо переключите роль на локальную модель.

## Конвейер и Демон

**Агенты «не видят» файлы проекта**
Проверьте поле **«Путь к проекту»** в настройках — оно должно указывать на нужную папку.

**Демон не стартует / «уже запущен»**
Проверьте файл `auto_tasks\.daemon.pid`. Если процесс мёртв, а файл остался — удалите его и запустите снова. Только один экземпляр Демона работает одновременно.

**Задача ушла в `failed`**
Откройте файл задачи в `auto_tasks\failed\` — там поле `failed_reason` с причиной. При повторных провалах Оракул добавляет техническую подсказку к следующей попытке.

## AI Cleaner

**Случайно удалил нужное**
Ничего не потеряно: нажмите **↩️ Отменить** в сообщении об удалении или откройте **📦 Статус карантина** и восстановите сессию. Файлы лежат в `.ai_cleaner_quarantine` на соответствующем диске, пока вы не очистите карантин навсегда.

**Дубликаты находятся неверно**
Убедитесь, что включён тумблер **«Полный хэш дубликатов»** — он подтверждает совпадение полным MD5, а не только по краям файла.

## Куда смотреть логи

Включите тумблер **«Режим отладки»** в настройках — появятся подробные логи в консоли. Демон пишет в свою консоль с префиксом `[DAEMON]`. API-ключи в логах автоматически маскируются.

Если проблема осталась — соберите текст ошибки из консоли и опишите шаги воспроизведения.

## Известные ограничения

Это осознанные компромиссы текущей версии, а не баги:

- **Выполнение терминальных команд (`shell=False`, без оболочки).** В режиме Конвейер агент может предложить терминальную команду; она выполняется без оболочки (`shell=False`) только после вашего подтверждения, что блокирует shell-инъекции через промпт. В автономном Демоне терминал заблокирован. Это локальный однопользовательский инструмент — не запускайте его на недоверенных задачах без присмотра.
- **Определение VRAM занижается для карт >4 ГБ.** Windows WMI отдаёт объём видеопамяти 32-битным числом, поэтому для RX 6600 (8 ГБ) автоопределение покажет ~4 ГБ. Менеджер моделей позволяет ввести точный объём вручную.
- **Локальные модели 7–8B и строгий JSON.** Маленькие модели иногда отклоняются от требуемого формата ответа агента. Парсеры устойчивы к этому, но при систематических сбоях используйте модель покрупнее для ролей Архитектор/Аудитор или облачный провайдер.
- **Только Windows.** AI Cleaner (реестр, буквы дисков) и определение VRAM рассчитаны на Windows. Конвейер и Менеджер работают кроссплатформенно, но официально проект ориентирован на Windows 10/11.

## Система забивает оперативную память (RAM) до 10-11 ГБ при загрузке модели

Симптом: модель весом ~5 ГБ загружена на видеокарту (VRAM занята), но при этом
системная оперативная память (RAM) забита на 10-11 ГБ, система начинает свопить и
тормозить. Особенно заметно на конфигурациях с 16 ГБ RAM.

Причина: технология **mmap** (memory-mapped files) в llama.cpp/LM Studio. По умолчанию
она отображает весь файл модели в RAM для быстрого доступа — даже если все слои
выгружены на GPU. В результате модель занимает память дважды: ~5 ГБ в VRAM плюс ~5 ГБ
в RAM. Плюс память на контекст и саму ОС — отсюда 10-11 ГБ.

Решение:
- **LM Studio:** откройте настройки загрузки модели и снимите галочку
  **«Use mmap (Keep model in RAM)»**. После этого веса будут жить только в VRAM.
- **Наш встроенный llama.cpp-сервер** (вкладка «Llama-сервер»): уже запускается с
  флагом `--no-mmap` по умолчанию — отдельных действий не требуется.
- **Контекст:** для систем с 8 ГБ VRAM держите размер контекста (-c) в районе 4096.
  Длинный контекст (16k-32k) при 8 ГБ VRAM почти гарантированно приводит к нехватке
  памяти (OOM). Наш сервер по умолчанию использует 4096.
