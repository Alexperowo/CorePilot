# Настройка CorePilot

Все настройки задаются в интерфейсе (⚙️ вверху) и сохраняются в файл `.ai_session.json` рядом с приложением. Облачные ключи API хранятся отдельно в `secrets.toml`.

---

## Облачные ключи API

Если хотите использовать облачные модели, создайте файл `secrets.toml` рядом с кодом проекта. Формат:

```toml
[PROVIDER_KEYS]
# Бесплатные провайдеры (рекомендуется для старта):
GROQ_API_KEY       = "ваш-ключ"
OPENROUTER_API_KEYS = "ваш-ключ"
CEREBRAS_API_KEY   = "ваш-ключ"
SAMBANOVA_API_KEY  = "ваш-ключ"
HF_TOKEN           = "hf_ваш-токен"
COHERE_API_KEY     = "ваш-ключ"
# Платные / с лимитами:
GEMINI_API_KEY     = "ваш-ключ"
OPENAI_API_KEY     = "ваш-ключ"
ANTHROPIC_API_KEY  = "ваш-ключ"
MISTRAL_API_KEY    = "ваш-ключ"
DEEPSEEK_API_KEY   = "ваш-ключ"
```

**Несколько ключей одного провайдера** (для обхода лимитов) — через запятую или списком:
```toml
GROQ_KEYS = "ключ1,ключ2,ключ3"
HF_TOKEN  = "hf_первый,hf_второй"
```
CorePilot автоматически чередует их при достижении лимита (round-robin). Если основной провайдер недоступен целиком, система переключается на резервный (межпровайдерный fallback) — полезно для ночных прогонов Демона.

> Имена ключей гибкие: HuggingFace распознаётся под `HF_TOKEN`, `HUGGINGFACE_API_KEY` или `HUGGINGFACEHUB_API_TOKEN`. OpenRouter — `OPENROUTER_API_KEYS` (с суффиксом S). Файл `secrets.toml` содержит секреты — не добавляйте его в Git. Если используете только локальные модели, этот файл не нужен.

## Каскад ролей (5 агентов)

Вверху настроек — по три-четыре выпадающих списка на каждую роль. Логика одинаковая:

```
Источник (local / cloud)
   ├─ local  → Бэкенд → Модель
   └─ cloud  → Провайдер → Модель
```

| Роль | Назначение | Совет |
|---|---|---|
| 🔍 **Сборщик** | изучает проект, собирает контекст | подойдёт лёгкая быстрая модель |
| 🏛 **Архитектор** | строит план изменений | модель посильнее (рассуждения) |
| 💻 **Кодер** | пишет код | модель, заточенная под код |
| 🔬 **Аудитор** | проверяет результат | средняя модель |
| 🔮 **Оракул** | подсказывает при провалах в Демоне | по умолчанию облако (Groq) |

Списки моделей подтягиваются автоматически: для `local` — с запущенного сервера (если сервер выключен, увидите пометку «бэкенд офлайн»); для `cloud` — актуальный список провайдера.

## Локальные бэкенды и порты

| Бэкенд | Порт по умолчанию | URL |
|---|---|---|
| LM Studio | 1234 | `http://localhost:1234/v1` |
| Ollama | 11434 | `http://localhost:11434/v1` |
| LlamaCPP (портативная версия) | 8080 | `http://localhost:8080/v1` |
| Lemonade | 8000 | `http://localhost:8000/v1` |

URL можно переопределить полем **«URL локального бэкенда»**.

## Важные тумблеры (все настраиваемые, без хардкода)

| Тумблер | По умолчанию | Зачем |
|---|---|---|
| 🧹 **Выгружать модель из VRAM между агентами** | выкл. | Для 8 ГБ VRAM при разных локальных моделях на роль. Выгружает модель завершившегося агента (только Ollama) перед следующим, чтобы две не висели в памяти. |
| 🔐 **Полный хэш дубликатов** | вкл. | Подтверждает дубликаты полным MD5 (надёжно). Выключение ускоряет, но возможны ложные совпадения. |
| 💽 **Карантин на диске источника** | вкл. | Карантин создаётся на том же диске, что и удаляемый файл (мгновенно на SSD). |
| 💾 **Авто-применение патчей** | выкл. | Применять изменения без подтверждения diff. Включайте осознанно. |
| 🔒 **Строгая песочница** | вкл. | Изменения сначала во временную копию, потом — в проект. |
| 🔮 **Мастер-Оракул** | вкл. | Подсказки при повторных провалах задач в Демоне. |
| 🌐 **Доступ в интернет** | выкл. | Веб-поиск для агентов. |
| 🐛 **Режим отладки** | выкл. | Подробные логи. |

## Прочие параметры

- **Путь к проекту** — папка, с которой работают агенты.
- **Стек агентов** — языковой профиль (Python, JS, C++ и т.д.) или авто-определение.
- **Скорость / качество** — `fast` / `medium` / `slow` (влияет на «температуру» моделей).
- **Слайдеры лимитов** — макс. итераций агента, запросов в минуту, размер файла, размер вывода инструментов, лимит дерева проекта, **таймаут шага агента** (по умолчанию 300 с — увеличьте, если медленная локальная модель не успевает на этапе Кодера).
