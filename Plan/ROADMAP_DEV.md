# ROADMAP_DEV.md — Приоритеты разработки

## Приоритет 1: Мобильное не видит модели (Android 11+)

### Статус: КОД ГОТОВ, тест на эмуляторе/устройстве не выполнен

**Проблема:** `list_models()` молча возвращал `[]` при OSError (нет прав на хранилище).
На Android 11+ для папки `Download/CorePilot/models` нужно `MANAGE_EXTERNAL_STORAGE`.

**Что сделано:**
- [x] `android_bridge.list_models()` → возвращает `[{"error": True, "reason": "..."}]` при ошибке
- [x] `android_bridge.check_manage_external_storage()` — проверяет разрешение через JNI
- [x] `android_bridge.request_manage_external_storage()` — открывает экран выдачи прав
- [x] `mobile/main.py` Edge AI вкладка — блок с ошибкой + кнопка «Выдать доступ» при permission error

**Нужно протестировать (живая система):**
- [ ] APK установить на эмулятор/устройство
- [ ] Проверить: при отсутствии прав — появляется кнопка, текст понятный
- [ ] Выдать права → список моделей появляется
- [ ] При `No such file` — понятное сообщение (папка не создана)

## Приоритет 3: Полировка трёх режимов

### Статус: НЕ ВЫПОЛНЕНО, требует живой системы

**3.1 AI Cleaner — полный цикл:**
- [ ] scan → отчёт с группами safe/warn/danger
- [ ] карантин → подтверждение → перемещение
- [ ] восстановление из карантина (undo)
- [ ] удаление навсегда
- [ ] просмотр карантина (список сессий)

**3.2 Конвейер — полный прогон на реальном коде:**
- [ ] Задача с реальным файлом проекта
- [ ] Все 4 агента (gatherer → architect → coder → auditor) отрабатывают
- [ ] Вердикт «УТВЕРЖДЕНО» / «ОТКЛОНЕНО» корректно отображается
- [ ] Checkpoint работает (прерви на середине → resume)

**3.3 Демон — ночной сценарий:**
- [ ] Создать 3+ задачи в очереди (в т.ч. DAG-зависимость)
- [ ] Запустить Демон → остановить → убедиться что задачи доделаны
- [ ] Замороженная задача (upstream failed) отображается правильно
- [ ] requeue застрявшей задачи работает
