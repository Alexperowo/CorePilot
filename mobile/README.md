# CorePilot Mobile

Приложение-компаньон для Android к десктопной фабрике CorePilot. Совмещает две роли:

- **Edge AI Node** — запускает тяжёлые GGUF-модели локально на телефоне (через llama.cpp). Прицел на флагманы с 24 ГБ LPDDR5X, где модель целиком помещается в память.
- **Пульт управления ПК** — голосовой и текстовый чат с агентом: либо с локальной моделью на телефоне, либо с десктопным CorePilot по сети.

Интерфейс на [Flet](https://flet.dev) (Python): тёмная высококонтрастная тема, нижний `NavigationBar` на 4 раздела — **Edge AI · Чат · Логи · Настройки**.

---

## Быстрый старт (для простого пользователя)

Четыре двойных клика на ПК — и приложение на телефоне:

1. **`setup_engine.bat`** — скачивает движок llama.cpp для Android (однократно; обновление — `update_engine.bat`).
2. **`build_apk.bat`** — собирает APK с движком внутри. Нужен только Python 3.12; Flutter SDK, JDK и Android SDK Flet скачает сам при первой сборке (~1-2 ГБ, 10-20 минут однократно).
3. **`install_apk.bat`** — ставит собранный APK на подключённый телефон (через adb).
4. **`update_mobile.bat`** — заливает модели `.gguf` на телефон.

Посмотреть интерфейс на ПК без сборки: **`run_dev.bat`**.


Батники:

| Батник | Что делает |
|--------|-----------|
| `setup_engine.bat` | **Первое**: скачивает движок llama.cpp для Android и прописывает в проект |
| `build_apk.bat` | Сборка APK (ставит Flet, проверяет движок, запускает `flet build apk`) |
| `install_apk.bat` | Установка готового APK на телефон через adb |
| `update_mobile.bat` | Заливка моделей `.gguf` на телефон |
| `update_engine.bat` | Обновление движка llama.cpp до последней версии (бэкап + скачивание) |
| `run_dev.bat` | Предпросмотр приложения на ПК (без сборки) |

## Архитектура (важное ограничение Android)

На нерутованном Android **нельзя исполнить бинарник из публичной папки**: внешнее хранилище (`Download`) смонтировано `noexec`, а домашняя папка приложения запрещает запуск нативного кода правилом W^X (для targetAPI ≥ 29). Единственное исполняемое место — каталог нативных библиотек приложения (`nativeLibraryDir`, read-only, но executable).

Поэтому:

```
APK (нативные библиотеки, nativeLibraryDir):
  libllama-server.so   ← движок, исполняется отсюда
  libllama.so, libggml*.so, ...  ← его зависимости

/storage/emulated/0/Download/CorePilot/   (публичная папка):
  └─ models/   ← файлы .gguf (читаются обычным open(), НЕ исполняются)
```

- **Движок** встроен в APK и запускается из `nativeLibraryDir` с `LD_LIBRARY_PATH`. Обновление движка = новая сборка APK.
- **Модели** `.gguf` лежат в публичной папке и обновляются с ПК через `adb push` (`update_mobile.bat`) — их не нужно исполнять, только читать.

> Почему так, а не «бинарник в Download + adb push»: такой путь физически не запустится на стоковом Android. Встраивание движка в APK — единственный поддерживаемый способ on-device инференса без рута.

Платформенные вызовы (pyjnius, subprocess, пути) изолированы в `android_bridge.py`. На ПК приложение запускается с фолбэками — можно разрабатывать UI без устройства.

## Разработка (на ПК, без устройства)

```bat
pip install flet
flet run
```

Откроется окно приложения. Голос и локальный сервер работают в режиме фолбэка (TTS печатает в консоль, STT сообщает, что доступен только на Android) — но вся навигация, настройки и связь с ПК-сервером по сети работают.

---

## Сборка APK

Требуется Flutter SDK (Flet скачает автоматически при первой сборке).

```bat
flet build apk
```

Готовый `.apk` появится в `build/apk/`. Установите на телефон.

---

## Развёртывание

**Движок** (llama-server + его `.so`, собранные под `arm64-v8a`) встраивается в APK при сборке — см. секцию `[tool.flet.android]` в `pyproject.toml`. Обновление движка = пересобрать APK (`build_apk.bat`) и переустановить (`install_apk.bat`).

**Модели** заливаются с ПК отдельно:

1. На ПК рядом с `update_mobile.bat` создайте папку `models\` и положите туда `.gguf`.
2. Подключите телефон по USB («Отладка по USB») **или** по Wi-Fi (`adb connect IP:5555`).
3. Запустите `update_mobile.bat` — он зальёт модели в `Download/CorePilot/models` на телефоне.
4. В приложении: вкладка **Edge AI** → выберите модель → **Запустить сервер**.

> Где взять движок под Android: соберите llama.cpp с NDK под `arm64-v8a` (получите `llama-server` + `libllama.so`, `libggml*.so` и др.), переименуйте исполняемый файл в `libllama-server.so` и пропишите все `.so` в `[tool.flet.android]`. Модель исполнять не нужно — она читается как файл.

## Голосовой ввод/вывод

- **Микрофон** на вкладке «Чат» — системное распознавание речи Android (через `RecognizerIntent`). Распознанный текст автоматически отправляется агенту.
- **Озвучка ответов** — системный TTS (`android.speech.tts.TextToSpeech`). Включается/выключается тумблером в «Настройках».

При первом запуске приложение запросит разрешение на микрофон.

---

## Связь с десктопным CorePilot

В «Настройках» укажите IP-адрес домашнего ПК и порт (по умолчанию 8080). Если локальный сервер на телефоне не запущен, чат обращается к ПК по OpenAI-совместимому API (`/v1/chat/completions`). Если запущен — работает полностью на телефоне, без сети.
